FinResilience Pro Security Protocol

We treat the security of your financial data as our highest priority. Below is an overview of the systems and procedures we employ to keep your data safe and private.

Core Security Pillars

[PILLAR 01] - DATA ENCRYPTION

All data transmitted between your browser and our servers is encrypted in transit using Transport Layer Security (TLS 1.3). Sensitive database fields and user passwords are encrypted at rest using industry-standard hashing algorithms (bcrypt/AES-256).

[PILLAR 02] - ISOLATED ENVIRONMENT

Our platform logic and database nodes run in secure, isolated containers. PocketBase migrations are run automatically using version-controlled, isolated steps to prevent accidental data leaks or unauthorized cross-contamination.

[PILLAR 03] - NO DIRECT BANK PASSWORDS

FinResilience Pro does not ask for, store, or access your raw bank passwords or direct credentials. We rely on manual input or secure aggregated snapshots to model calculations, removing the risk of bank credential theft.

Reporting Vulnerabilities

If you identify a security vulnerability within our platform, please do not disclose it publicly. Contact our security team immediately at security@finresiliencepro.local to report your findings. We investigate all valid reports within 48 hours.
